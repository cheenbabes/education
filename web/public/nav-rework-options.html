Nav rework — PM review

The "Planner" dropdown feels awkward. Here's the diagnosis and four reworks — each with pros, cons, and a preview of the live nav chrome.

Why "Planner" feels off

  • "Planner" is a category with no mental model. Dashboard is a home/overview. Lessons is a searchable library. Standards is a reference tool. These three aren't united by a clear user story — "Planner" reads like engineering naming, not homeschool-parent naming.
  • 3-item dropdowns cost clicks without hiding real complexity. Dropdowns earn their keep at 5+ items or true sub-nav. Surfacing three load-bearing pages behind one hover = friction tax.
  • Dashboard and Lessons overlap in intent. Dashboard already shows "Your lessons." A separate Lessons page exists for filtering/search, but a new user seeing both in a dropdown wonders which is "the" lessons page.
  • Create deserves emphasis. It's the funnel. "Upgrade" is already pinned on the right; Create should be first-class left, visually prominent. Today it's just a text link next to the dropdown.

Option 1 · FlattenMinimal change

Kill the Planner dropdown. Put Dashboard, Lessons, Standards as separate top-level tabs. Keep everything else identical.

Pros

  • Every click goes somewhere useful — no hover-to-reveal
  • Matches mental model: each page is its own thing
  • Smallest code diff

Cons

  • Nav goes from 8 items to 9 — slightly denser
  • Dashboard/Lessons overlap still present
  • Mobile: starts to wrap

Option 2 · ConsolidateRecommended

Merge Dashboard and Lessons into a single Home page. Dashboard stays what it is today; Lessons becomes a tab inside it (or a filter: "All", "Favorites", "Completed"). Promote Standards and Create to top-level.

Pros

  • Kills the IA redundancy — one source of truth for "my stuff"
  • Nav drops to 6 items — airy and clear
  • Home is a universal mental model; no naming debate
  • Forces us to make Dashboard actually good (no more lazy links)

Cons

  • One-time work: move /lessons table into /dashboard as a tab
  • Power users with lots of lessons need good in-page filtering
  • Old /lessons URLs need a redirect

Option 3 · Rename & flattenNo IA work

Keep Dashboard and Lessons as separate top-level pages but rename them to something concrete. Dashboard → Home. Lessons stays. Remove "Planner" entirely.

Pros

  • No code refactoring — just rename and unnest
  • "Home" is universally understood; "Dashboard" is biz-jargon
  • Keeps /lessons for users who filter/search heavily

  • Home vs Lessons overlap still present
  • Doesn't solve the "two places to see lessons" problem
  • Nav at 8 items — middle density

Option 4 · Group by user jobAmbitious

Reorganize the nav around what parents are actually trying to do. Teach (Home, Lessons, Standards, Compass). Explore (Archetypes, Explore, About). Account stays right-side with Upgrade. Still two dropdowns — but each is a coherent category.

Pros

  • Category names actually mean something to a parent
  • Nav looks clean: 3 labels + Create + avatar
  • Room to grow (Tools, Community) without redesigning

Cons

  • Still uses dropdowns — same friction as today
  • Over-engineered for current 8-page surface
  • Parents might not grok "Teach" vs "Explore" boundary

My take: Option 2 is the right long game. "Planner" is a symptom of unclear IA around Dashboard-vs-Lessons. Fixing the naming (Options 1 & 3) doesn't fix the confusion — you still have two places that show "your lessons." Option 2 forces us to make Home the single planning surface and kills the ambiguity for good. It also ships a cleaner 6-item nav, leaves room for future expansion, and keeps Create prominently on the left where it funnels new users.

If you want a fast win: ship Option 1 now (5 minutes of work — just delete the dropdown, flatten three links), and commit to Option 2 as a next sprint. That ships the user benefit immediately without having to finish the refactor today.
